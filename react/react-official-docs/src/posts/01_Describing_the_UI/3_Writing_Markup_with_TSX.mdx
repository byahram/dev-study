# Writing Markup with JSX

## 📌 JSX란?

JSX(JavaScript XML)는 **JavaScript 코드 안에서 HTML과 유사한 문법을 사용하여 UI를 정의하는 문법**입니다. 이를 통해 코드가 더 직관적으로 보이며, `React.createElement()`를 사용하는 것보다 가독성이 높아집니다.

```jsx
export default function Hello() {
  return <h1>Hello, world!</h1>;
}
```

<br />

## 📝 JSX에서 JavaScript 표현식 사용하기

JSX 내부에서 `{}`를 사용하면 **JavaScript 표현식**을 삽입할 수 있습니다.

### 🔹 변수 사용

```jsx
const name = "Ahram";
return <h1>Hello, {name}!</h1>;
```

### 🔹 함수 사용

```jsx
function formatName(user) {
  return user.firstName + " " + user.lastName;
}

const user = { firstName: "Kim", lastName: "Ahram" };
return <h1>Hello, {formatName(user)}!</h1>;
```

## 📌 여러 개의 요소 반환하기

JSX는 하나의 **루트 요소**만 반환해야 합니다. 여러 요소를 반환하려면 `<div>` 또는 `<Fragment>`를 사용해야 합니다.

### 🔹 Fragment 사용 (불필요한 태그 제거)

```jsx
return (
  <>
    <h1>Welcome</h1>
    <p>React를 배우는 중입니다.</p>
  </>
);
```

## 🎨 JSX의 속성 (Props)

JSX에서는 **HTML 속성처럼 속성을 설정**할 수 있지만, React에서는 **camelCase 표기법**을 사용해야 합니다.

### 🔹 속성 사용 예제

```jsx
const imageUrl = "https://example.com/image.png";
return <img src={imageUrl} alt="Example" />;
```

### 🔹 HTML과 다른 속성명 비교

| HTML      | JSX         |
| --------- | ----------- |
| `class`   | `className` |
| `for`     | `htmlFor`   |
| `onclick` | `onClick`   |

## 📌 JSX 문법 주의사항

1. 모든 태그는 **닫혀 있어야 합니다.**
2. **셀프 클로징 태그**는 `<img />`, `<input />`처럼 `/`를 붙여야 합니다.

### 🔹 올바른 JSX 문법 예제

```jsx
return (
  <div>
    <h1>제목</h1>
    <p>내용입니다.</p>
  </div>
);
```

## 🚀 추가 팁

- **JSX와 일반 HTML의 차이점**을 이해하면 더 쉽게 사용할 수 있습니다.
- **React DevTools**를 활용해 JSX 구조를 확인하면 디버깅이 편리합니다.
- 직접 JSX 코드를 작성하고 실행해보면 더 쉽게 익숙해질 수 있습니다.
